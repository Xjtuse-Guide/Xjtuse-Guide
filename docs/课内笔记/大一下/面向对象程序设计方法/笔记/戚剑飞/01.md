## 01

### Java 的用途
- Java 可用于构建程序和软件组件
- 程序是独立的实体，可以在 Java 虚拟机上运行

### 编译与解释并存
源代码由 Java 编译器编译为 Java 虚拟机（JVM）专用的机器码，这种 JVM 的机器码也被称为字节码， Java 虚拟机的解释器对指令进行解释和执行。字节码指令是架构中立的，因为它们运行在 Java 虚拟机（JVM）上，并不依赖于特定的硬件架构

同一个应用程序可以在所有平台上运行，只要该平台安装了 Java 虚拟机。在源代码级和二进制级都是可移植的，一份源代码会被编译成一组供 JVM 使用的字节码指令，无需重新编译即可在任何平台和架构上运行。

### 应用程序
一个独立的程序，可以访问系统资源，如文件。
不需要在网页浏览器中运行。通过命令行或菜单选择显式调用。
`main()` 方法是应用程序的入口点。

### 服务端程序（Servlet）
- 处理来自网页浏览器的请求并返回响应
- 在服务器端创建动态内容
- 运行在应用服务器内

### Java 服务器页面（JSP）
- 将 Java 代码嵌入的 HTML 页面
- 在服务器端创建动态内容，而不是在浏览器端
- 运行在应用服务器内

### JavaBeans
- 一种 Java 代码，具有其属性、方法和事件，旨在促进开发者之间的重用
- 可重载的软件组件，可以在构建工具中进行可视化操作

### 企业 JavaBeans（EJB）
- 分布式对象，允许不同 JVM 中的 Java 对象之间进行通信
- 封装应用程序的业务逻辑和模型
- 运行在应用服务器内

**1. 什么是 JVM？它的作用是什么？**
Java 虚拟机（JVM）是一种运行时环境，它负责执行 Java 程序中编译生成的字节码。它屏蔽了底层操作系统和硬件的差异，使得 Java 程序可以在不同的平台上运行而无需修改。

**2. 什么是字节码？**
字节码是一种中间代码，由 Java 编译器将源代码编译而成，它不是直接针对某种具体的硬件平台，而是由 JVM 解释执行。字节码的可移植性使 Java 实现了“编写一次，到处运行”的目标。

**3. 程序和组件的区别是什么？**
程序是可以独立运行的完整实体，例如应用程序或 Applet；而组件是构建程序的模块化部分，比如 Servlet、JSP、JavaBeans 或 EJB，它们通常需要运行在特定的容器或服务器中，不能单独运行。
