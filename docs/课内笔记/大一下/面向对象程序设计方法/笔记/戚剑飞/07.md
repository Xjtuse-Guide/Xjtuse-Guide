## 07

### 接口
- 在 Java 中，声明的类型可以是类或接口
- 接口代表了实现该接口的对象所支持的服务的承诺
- 接口代表了对象实现的契约
- 接口仅仅是方法声明的列表
- 它的方法是声明但未实现的
- 接口不是类，尽管它看起来与抽象类非常相似

- 在类声明中，超类的命名位于类支持的任何接口之前
- 如果一个类实现多个接口，这些接口会列出，并用逗号分隔

### 实现接口
- 一个实现接口的类必须实现该接口定义的每个方法
- 如果未实现一个或多个方法，编译器将产生错误
- 子类会自动实现其父类实现的所有接口
- 有些接口，如 `Cloneable`，并不包含任何方法
- 这些接口作为信号告诉 Java，某个类遵循特定协议，但该协议可能无法通过一组具体的方法来表达
- 以 `Cloneable` 为例，它提醒 JVM，实现该接口的类的对象可以通过 `clone()` 方法进行复制

### 类型和接口
- 变量的类型可以是接口
- 只有实现该接口的类的对象可以绑定到该变量
- 只能使用接口中定义的消息
- 接口不能出现在 `new` 表达式中

### 子接口
- 接口可以被扩展
- 接口不是类，因此这个层次结构独立于类层次结构
- 扩展另一个接口的接口会继承它的所有方法声明

### 协议
- 接口定义了一种协议（即一组方法）
- 如果一个类实现了某个接口，那么该类就实现了该协议
- 接口可以用来对一组没有继承关系的类强制执行一个共同的协议
- 在类继承链中，公共协议通过子类化来强制执行

### 使用接口的更多优势
- 使用接口可以更好地控制对象的使用方式
- 不同类之间的相似性可以通过接口来捕获，而不必强行建立不自然的继承关系
- 开发者可以知道对象会响应哪些消息，接口可以在不暴露类的情况下展示对象的行为
- 提高了代码的重用性
