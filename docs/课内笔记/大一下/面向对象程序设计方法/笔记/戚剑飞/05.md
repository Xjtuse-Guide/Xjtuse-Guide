## 05

### Classes
- 封装属性（字段）和行为（方法）
- 属性和行为是类的成员

**成员可以属于以下任一类型：**
- 整个类（类变量和方法，用关键字 `static` 表示）
- 单个对象（实例变量和方法）

**类可以是：**
- 相互独立的
- 通过继承相关（超类/子类）
- 通过类型相关（接口）

### 抽象类（abstract）
可以包含普通类中能包含的所有内容（变量、方法、构造器）
- 抽象类不能被实例化，只能被继承
- 它为子类提供通用的信息

### 构造方法没有返回类型
- 构造方法本身不返回值，但与 `new` 关键字一起使用时，会返回对新对象的引用

### 默认构造方法
- 没有参数的构造方法被称为默认构造方法
- 如果你没有显式定义任何构造方法，Java 平台会自动提供一个默认构造方法
- 当你定义了构造方法时，应该同时提供一个默认构造方法

### 构造方法的链式调用
- 父类对象会先于子类对象被构造
- 使用 `super(参数列表)` 来初始化父类的成员
- 构造方法的第一行只能是以下之一：
    - `super(参数列表);`
    - `this(参数列表);`
- 不能在同一个构造方法中同时使用 `super()` 和 `this()`
- 编译器会为所有构造方法自动提供一个隐式的 `super()` 调用

### 字段（Fields）
- 对象通过字段来保留状态
- 字段是在类定义中声明的
- 每个实例都有自己的一份实例变量副本
- 字段可以在声明时（如果需要）进行初始化
- 如果字段没有初始化，将使用默认值

### 方法签名
- 一个类可以有多个同名的方法
- 每个方法必须具有不同的签名

**方法签名由以下部分组成：**
- 方法名
- 参数的数量和类型

### 方法参数
参数（也称为实参）传递方式如下：
- 对于基本类型，是按值传递
- 对于引用类型，是按对象引用传递
在作为参数传递时，基本类型的值是不能被修改的

### 方法的重载
- 只要方法具有不同的签名，就可以使用相同的名称定义多个不同的方法

### 静态成员
- 静态字段和方法属于类本身。
- 在该类的一个对象中更改某个值，会影响所有该类的对象。
- 静态方法和字段可以在不实例化类的情况下访问。
- 静态方法和字段通过使用 `static` 关键字进行声明。

### Final成员
- 一个 Final 字段是不能被修改的字段。
- 这是 Java 中常量的实现方式。
- 通常，类相关的常量会被声明为静态最终字段，以便于访问。
- 一个常见的约定是使用全大写字母来命名这些常量。

### 抽象类
- 抽象类不能被实例化；它们旨在作为其他类的超类。
- 抽象方法没有实现。
- 如果一个类有一个或多个抽象方法，它就是抽象类，必须声明为抽象类。
- 具体类有完整的实现，可以被实例化。

### 包（Package）
类可以按以下方式分组：
- 根据你正在构建的模型逻辑地分组
- 作为一起使用的集合
- 为了方便
按照约定，包名使用小写字母
- 不同的包可以包含同名的类

### 类的可见性
- 类可以通过类名引用同一包内的其他类
- 类必须提供完全限定名（包括包名）来引用定义在不同包中的类

### 类的层次结构
- 每个对象都属于某个类
- 属于某个类的对象称为该类的实例
- 每个类（除了 `Object` 类）都有一个父类
- 在 Java 中，`Object` 是整个类层次结构的根类
- 当定义新类时，开发者必须决定哪个类是适合的父类
